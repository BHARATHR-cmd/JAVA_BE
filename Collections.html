Collections

  • What are the different types of Collections in Java?

    Java Collections Framework provides various types of collections to store and manipulate groups of objects. The main types of collections are:

    • List: An ordered collection that allows duplicate elements. Examples include ArrayList, LinkedList, and Vector.
    • Set: A collection that does not allow duplicate elements. Examples include HashSet, LinkedHashSet, and TreeSet.
    • Queue: A collection used to hold multiple elements prior to processing. Examples include LinkedList, PriorityQueue, and ArrayDeque.
    • Map: A collection that maps keys to values, with no duplicate keys allowed. Examples include HashMap, LinkedHashMap, and TreeMap.

  • What is a List in Java?
    A List in Java is an ordered collection that allows duplicate elements. It provides positional access and insertion of elements. Examples include ArrayList, LinkedList, and Vector.
    
        // Example of using a List in Java
        List list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Apple"); // Allows duplicate elements
        System.out.println(list); // Output: [Apple, Banana, Apple]
    Real-life scenario: A List can be used to store a list of items in a shopping cart where duplicate items are allowed.
  • When should you use an ArrayList in Java?
    You should use an ArrayList when you need fast random access to elements and when the size of the list is not expected to change frequently. ArrayList provides constant-time positional access and is backed by an array.
    
        // Example of using an ArrayList in Java
        List arrayList = new ArrayList<>();
        arrayList.add("Apple");
        arrayList.add("Banana");
        System.out.println(arrayList.get(1)); // Output: Banana
    Real-life scenario: An ArrayList can be used to store a list of students in a class where the list is not expected to change frequently.
  • When should you use a LinkedList in Java?
    You should use a LinkedList when you need fast insertion and deletion of elements at both ends of the list. LinkedList provides constant-time insertions and deletions and is implemented as a doubly-linked list.
    
        // Example of using a LinkedList in Java
        List linkedList = new LinkedList<>();
        linkedList.add("Apple");
        linkedList.addFirst("Banana");
        linkedList.addLast("Cherry");
        System.out.println(linkedList); // Output: [Banana, Apple, Cherry]
    Real-life scenario: A LinkedList can be used to implement a queue for a printer where documents are added and removed from both ends.
  • What is a Set in Java?
    A Set in Java is a collection that does not allow duplicate elements. It models the mathematical set abstraction and provides methods for set operations. Examples include HashSet, LinkedHashSet, and TreeSet.
    
        // Example of using a Set in Java
        Set set = new HashSet<>();
        set.add("Apple");
        set.add("Banana");
        set.add("Apple"); // Duplicate element, will not be added
        System.out.println(set); // Output: [Apple, Banana]
    Real-life scenario: A Set can be used to store a collection of unique email addresses for a newsletter subscription.
  • When should you use a HashSet in Java?
    You should use a HashSet when you need a collection that does not allow duplicate elements and when you do not care about the order of elements. HashSet provides constant-time performance for basic operations like add, remove, and contains.
    
        // Example of using a HashSet in Java
        Set hashSet = new HashSet<>();
        hashSet.add("Apple");
        hashSet.add("Banana");
        System.out.println(hashSet.contains("Apple")); // Output: true
    Real-life scenario: A HashSet can be used to store a collection of unique product IDs in an inventory system.
  • When should you use a LinkedHashSet in Java?
    You should use a LinkedHashSet when you need a collection that does not allow duplicate elements and when you need to maintain the insertion order of elements. LinkedHashSet provides predictable iteration order.
    
        // Example of using a LinkedHashSet in Java
        Set linkedHashSet = new LinkedHashSet<>();
        linkedHashSet.add("Apple");
        linkedHashSet.add("Banana");
        linkedHashSet.add("Cherry");
        System.out.println(linkedHashSet); // Output: [Apple, Banana, Cherry]
    Real-life scenario: A LinkedHashSet can be used to store a collection of unique user actions in the order they were performed.
  • When should you use a TreeSet in Java?
    You should use a TreeSet when you need a collection that does not allow duplicate elements and when you need the elements to be sorted in their natural order or by a custom comparator. TreeSet provides log(n) time cost for basic operations.
    
        // Example of using a TreeSet in Java
        Set treeSet = new TreeSet<>();
        treeSet.add("Banana");
        treeSet.add("Apple");
        treeSet.add("Cherry");
        System.out.println(treeSet); // Output: [Apple, Banana, Cherry]
    Real-life scenario: A TreeSet can be used to store a collection of unique words in a dictionary, sorted in alphabetical order.
  • What is a Queue in Java?
    A Queue in Java is a collection used to hold multiple elements prior to processing. It follows the First In First Out (FIFO) principle. Examples include LinkedList, PriorityQueue, and ArrayDeque.
    
        // Example of using a Queue in Java
        Queue queue = new LinkedList<>();
        queue.add("Apple");
        queue.add("Banana");
        System.out.println(queue.poll()); // Output: Apple
    Real-life scenario: A Queue can be used to manage tasks in a task scheduler where tasks are processed in the order they are added.
  • When should you use a LinkedList as a Queue in Java?
    You should use a LinkedList as a Queue when you need a collection that allows fast insertion and deletion of elements at both ends. LinkedList provides constant-time insertions and deletions and can be used as a deque (double-ended queue).
    
        // Example of using a LinkedList as a Queue in Java
        Queue linkedListQueue = new LinkedList<>();
        linkedListQueue.add("Apple");
        linkedListQueue.add("Banana");
        System.out.println(linkedListQueue.poll()); // Output: Apple
    Real-life scenario: A LinkedList can be used to implement a queue for customer service requests where requests are processed in the order they are received.
  • When should you use a PriorityQueue in Java?
    You should use a PriorityQueue when you need a collection that orders its elements according to their natural ordering or by a custom comparator. PriorityQueue provides log(n) time cost for the enqueuing and dequeuing methods.
    
        // Example of using a PriorityQueue in Java
        Queue priorityQueue = new PriorityQueue<>();
        priorityQueue.add(3);
        priorityQueue.add(1);
        priorityQueue.add(2);
        System.out.println(priorityQueue.poll()); // Output: 1
    Real-life scenario: A PriorityQueue can be used to manage tasks in a task scheduler where tasks with higher priority are processed first.
  • When should you use an ArrayDeque in Java?
    You should use an ArrayDeque when you need a resizable array implementation of a deque (double-ended queue). ArrayDeque provides constant-time performance for insertions and deletions at both ends and is faster than LinkedList for this purpose.
    
        // Example of using an ArrayDeque in Java
        Deque arrayDeque = new ArrayDeque<>();
        arrayDeque.addFirst("Apple");
        arrayDeque.addLast("Banana");
        System.out.println(arrayDeque.pollFirst()); // Output: Apple
    Real-life scenario: An ArrayDeque can be used to implement a deque for a browser's back and forward navigation history.
  • What is a Map in Java?
    A Map in Java is a collection that maps keys to values, with no duplicate keys allowed. It provides methods for basic operations like put, get, and remove. Examples include HashMap, LinkedHashMap, and TreeMap.
    
        // Example of using a Map in Java
        Map map = new HashMap<>();
        map.put("Apple", 1);
        map.put("Banana", 2);
        System.out.println(map.get("Apple")); // Output: 1
    Real-life scenario: A Map can be used to store a collection of product IDs and their corresponding quantities in an inventory system.
  • When should you use a HashMap in Java?
    You should use a HashMap when you need a collection that maps keys to values and when you do not care about the order of the elements. HashMap provides constant-time performance for basic operations like put, get, and remove.
    
        // Example of using a HashMap in Java

  • When should you use a LinkedHashMap in Java?
    You should use a LinkedHashMap when you need a collection that maps keys to values and when you need to maintain the insertion order of elements. LinkedHashMap provides predictable iteration order.
    
        // Example of using a LinkedHashMap in Java

  • When should you use a TreeMap in Java?
    You should use a TreeMap when you need a collection that maps keys to values and when you need the keys to be sorted in their natural order or by a custom comparator. TreeMap provides log(n) time cost for basic operations.
    
        // Example of using a TreeMap in Java
        Map treeMap = new TreeMap<>();

        treeMap.put("Apple", 1);
        System.out.println(treeMap); // Output: {Apple=1, Banana=2}
    Real-life scenario: A TreeMap can be used to store a collection of student names and their corresponding grades, sorted in alphabetical order.
